# Eval Results

Generated by `python eval/run_eval.py`. Each run writes a file named
`run_<YYYY-MM-DD>.json` with the configuration used and the four RAGAs
metrics:

```json
{
  "config": {
    "chunk_size": 512,
    "chunk_overlap": 64,
    "top_k": 5,
    "retrieval_mode": "hybrid",
    "embed_model": "...",
    "llm_model": "...",
    "eval_date": "..."
  },
  "metrics": {
    "faithfulness": 0.0,
    "answer_relevancy": 0.0,
    "context_recall": 0.0,
    "context_precision": 0.0
  },
  "timestamp": "..."
}
```

The companion script `eval/check_regression.py` reads the two most recent
results in this directory and exits non-zero if any metric drops by more
than `REGRESSION_THRESHOLD` (default 5%). Wire it into CI to fail builds
on retrieval-quality regressions.

Result files are not checked into git — they are produced per-run from
your local backend (Ollama, OpenAI, Azure, etc.). Set the LLM provider
via env vars before running:

```bash
export LLM_PROVIDER=openai LLM_MODEL=gpt-4o-mini LLM_API_KEY=...
export EMBED_PROVIDER=sentence-transformers EMBED_MODEL=all-MiniLM-L6-v2
python eval/run_eval.py
python eval/check_regression.py  # exits 1 if metrics dropped vs prior run
```
