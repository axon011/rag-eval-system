# .agent.md - rag-eval-system

## Project Info

Production RAG Q&A system with full evaluation pipeline using RAGAs metrics and MLflow experiment tracking. Features a modern web UI with support for multiple LLM providers.

## Key Technologies
- FastAPI (REST API + Web UI)
- Ollama (local LLM: llama3.2, nomic-embed-text)
- OpenRouter (external LLM: z-ai/glm-4.5-air:free, gpt-4o-mini, etc.)
- OpenAI & Anthropic support
- Qdrant (vector database)
- RAGAs (evaluation metrics)
- MLflow (experiment tracking)
- LangChain (RAG orchestration)

## Key Features
- **Web UI**: Query, Upload, and Settings tabs
- **Multi-Provider**: Ollama, OpenRouter, OpenAI, Anthropic
- **Hybrid Retrieval**: Dense (Qdrant) + Sparse (BM25) with RRF
- **Caching**: LRU cache for embeddings and queries
- **Evaluation**: RAGAs metrics with MLflow tracking

## Key Endpoints
- `GET /` - Web UI
- `GET /health` - Health check
- `POST /ingest/` - Upload PDF/MD/TXT
- `POST /query/` - Ask questions

## Running Services
- FastAPI: http://localhost:8000
- Qdrant: http://localhost:6333
- MLflow: http://localhost:5000
- Ollama: http://localhost:11434

## Common Commands
```bash

docker-compose up -d

# Rebuild after changes
docker-compose up -d --build fastapi

# View logs
docker logs rag-fastapi
```

## How to Identify This Project
- **Keywords**: RAG, Qdrant, Ollama, OpenRouter, evaluation, MLflow, web UI
- **Unique feature**: Has web UI + evaluation harness
