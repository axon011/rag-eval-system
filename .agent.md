# .agent.md - rag-eval-system

## Project Info

Production RAG Q&A system with full evaluation pipeline using RAGAs metrics and MLflow experiment tracking.

## Key Technologies
- FastAPI (REST API)
- Ollama (local LLM: llama3.2, nomic-embed-text)
- Qdrant (vector database)
- RAGAs (evaluation metrics)
- MLflow (experiment tracking)
- LangChain (RAG orchestration)

## Directory Structure

```
rag-eval-system/
├── app/
│   ├── main.py              # FastAPI entrypoint
│   ├── routes/
│   │   ├── ingest.py        # PDF upload & indexing
│   │   └── query.py         # Q&A endpoint
│   ├── core/
│   │   ├── embedder.py      # Ollama embeddings
│   │   ├── retriever.py     # Hybrid retrieval (dense + sparse)
│   │   ├── generator.py     # LLM generation
│   │   └── pipeline.py      # Full RAG chain
│   └── models/
│       └── schemas.py       # Pydantic models
├── eval/
│   ├── dataset.py           # Evaluation dataset
│   └── run_eval.py          # RAGAs evaluation
├── mlflow_tracking/
│   └── log_experiment.py    # MLflow logging
├── tests/
├── docker-compose.yml
├── Dockerfile
└── requirements.txt
```

## Key Endpoints
- `POST /ingest/` - Upload PDF document
- `POST /query/` - Ask questions

## How to Identify This Project
- **Keywords**: RAG, Qdrant, Ollama, evaluation, MLflow, RAGAs
- **Unique feature**: Has evaluation harness with RAGAs metrics
- **NOT**: This is NOT multi-agent-pipeline (that's LangGraph/CrewAI) or llmops-dashboard (that's React/Langfuse)
